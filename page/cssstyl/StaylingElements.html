position:

در CSS، ویژگی position برای تعیین نحوه مکان‌یابی یک عنصر در صفحه استفاده می‌شود. این ویژگی تعیین می‌کند که چگونه یک عنصر در رابطه با عناصر دیگر یا والدین خود جایگذاری شود.

مقادیر position:

1.static (پیش‌فرض):این مقدار پیش‌فرض است. عنصر بر اساس جریان عادی سند قرار می‌گیرد (بدون موقعیت خاص). نمی‌توان از ویژگی‌های top، right، bottom، و left استفاده کرد

}div
; position: static
{

2.relative:عنصر نسبت به موقعیت اصلی خود (جریان عادی سند) جابجا می‌شود. می‌توان از top، right، bottom، و left برای جابجایی استفاده کرد. فضای اولیه عنصر حفظ می‌شود.

} div
; position: relative
/* 20px به پایین جابجا می‌شود */;top: 20px
/* 10px به راست جابجا می‌شود */;left: 10px
{

3.absolute:عنصر نسبت به نزدیک‌ترین عنصر دارای position غیر از static (مثل relative, absolute, یا fixed) موقعیت‌دهی می‌شود. اگر عنصر والد مناسب وجود نداشته باشد، نسبت به کل صفحه (viewport) قرار می‌گیرد. از جریان عادی خارج می‌شود.

} div
;position: absolute
; top: 50px
; right: 20px
{

4.fixed:عنصر نسبت به کل صفحه (viewport) موقعیت‌دهی می‌شود و با اسکرول صفحه جابجا نمی‌شود. برای ساختن هدر یا نوارهای ثابت در صفحات وب استفاده می‌شود.

}div
; position: fixed
; bottom: 0
; right: 0
{

5.sticky:ترکیبی از رفتار relative و fixed است. عنصر تا زمانی که در محدوده مشخصی از والد خود باشد، مانند relative عمل می‌کند و وقتی به آستانه مشخصی می‌رسد، به حالت fixed درمی‌آید. برای استفاده باید حداقل یک مقدار top, bottom, left, یا right مشخص شود.

div {
; position: sticky
/* وقتی به بالای صفحه رسید، ثابت می‌شود */ ;top: 0
}

display:

در CSS، ویژگی display برای تعیین نحوه نمایش و رفتار یک عنصر در صفحه استفاده می‌شود. این ویژگی نقش مهمی در چیدمان عناصر دارد و مشخص می‌کند که یک عنصر چگونه باید با عناصر دیگر تعامل کند.

مقدارهای پرکاربرد display:

1.block:عنصر به‌صورت یک بلوک کامل نمایش داده می‌شود. عرض عنصر به‌طور پیش‌فرض تمام عرض والد خود را می‌گیرد. شروع و پایان آن یک خط جدید ایجاد می‌کند.

} div
;display: block
{

2.inline:عنصر به‌صورت درون‌خطی (inline) نمایش داده می‌شود. عرض و ارتفاع عنصر قابل تغییر نیست. بدون شکستن خط در کنار عناصر دیگر قرار می‌گیرد.

}span
;display: inline
{

3.inline-block:ترکیبی از رفتار inline و block است. در کنار عناصر دیگر در یک خط قرار می‌گیرد اما می‌توان عرض و ارتفاع آن را تغییر داد.

} button
; display: inline-block
; width: 100px
; height: 50px
{

4.none:عنصر به‌طور کامل از صفحه حذف می‌شود (هیچ فضایی اشغال نمی‌کند). در این حالت عنصر همچنان در DOM وجود دارد اما در صفحه نمایش داده نمی‌شود.

} div
; display: none
{

5.flex:

عنصر به یک container فلکس تبدیل می‌شود. برای چیدمان انعطاف‌پذیر عناصر داخلی استفاده می‌شود. مثال: چینش عناصر به‌صورت افقی یا عمودی.

}div
;display: flex
; justify-content: center
;align-items: center
{

6.grid:

عنصر به یک container شبکه‌ای (grid) تبدیل می‌شود. برای چیدمان پیشرفته به‌صورت سطر و ستون استفاده می‌شود.

}div
; display: grid
;grid-template-columns: repeat(3, 1fr)
{

7.inline-flex:

مشابه flex است اما رفتار inline دارد. به این معنا که در کنار عناصر دیگر در یک خط قرار می‌گیرد.

} div
;display: inline-flex
{

8.inline-grid:

مشابه grid است اما رفتار inline دارد. شبکه‌ای است که می‌تواند در یک خط با سایر عناصر قرار گیرد.

} div
;display: inline-grid
{

gap:

در CSS، ویژگی gap برای تنظیم فاصله بین آیتم‌ها در چیدمان‌های فلکس (Flex)، شبکه (Grid) یا ستون‌های چندگانه (Multi-column) استفاده می‌شود. این ویژگی به شما اجازه می‌دهد به راحتی فاصله میان عناصر داخلی را بدون نیاز به استفاده از مارجین‌های جداگانه کنترل کنید.


نحوه استفاده از gap:

gap در Flexbox و Grid در Flexbox فاصله بین آیتم‌ها در یک ردیف یا ستون تنظیم می‌شود. در Grid می‌توانید فاصله بین سطرها و ستون‌ها را به صورت جداگانه مدیریت کنید.

row-gap: فاصله بین ردیف‌ها.

column-gap: فاصله بین ستون‌ها.

اگر تنها یک مقدار تعریف کنید، هم برای سطرها و هم برای ستون‌ها اعمال می‌شود.

;gap: [row-gap] [column-gap]


مقادیر:

1-مقدار یک واحد اگر فقط یک مقدار تعریف کنید، برای هر دو (سطر و ستون) استفاده می‌شود.

;gap: 20px

2-دو مقدار می‌توانید فاصله سطرها و ستون‌ها را جداگانه تنظیم کنید:

/* 20px برای سطرها و 10px برای ستون‌ها */; gap: 20px 10px

نکات:

پشتیبانی مرورگرها:gap به خوبی در Grid و Flexbox پشتیبانی می‌شود، اما در نسخه‌های قدیمی‌تر مرورگرها (مخصوصاً در Flexbox) ممکن است کار نکند.
اگر مشکلی در پشتیبانی دارید، می‌توانید از مارجین به عنوان جایگزین استفاده کنید.
استفاده از gap باعث کاهش پیچیدگی کد شما می‌شود و نیازی به تنظیم دستی مارجین بین آیتم‌ها نیست.


flex-direction:

ویژگی flex-direction در CSS مشخص می‌کند که چگونه آیتم‌های یک کانتینر فلکس (Flexbox) در محور اصلی (main axis) مرتب شوند. این ویژگی جهت قرارگیری آیتم‌ها را تعیین می‌کند، مثلاً افقی یا عمودی.

مقادیر flex-direction:

1.row (پیش‌فرض):

آیتم‌ها در یک ردیف افقی مرتب می‌شوند.
محور اصلی: از چپ به راست (برای زبان‌های چپ به راست مثل انگلیسی).
محور متقاطع (Cross Axis): عمودی.

} container.
;display: flex
;flex-direction: row
{

2.row-reverse:

یتم‌ها در یک ردیف افقی مرتب می‌شوند، اما ترتیب آن‌ها معکوس است (از راست به چپ برای زبان‌های چپ به راست).
محور اصلی: از راست به چپ.

} container.
;display: flex
; flex-direction: row-reverse br {

3.column:

آیتم‌ها در یک ستون عمودی مرتب می‌شوند.
محور اصلی: از بالا به پایین.
محور متقاطع: افقی.

} container.
;display: flex
;flex-direction: column
{

4.column-reverse:

آیتم‌ها در یک ستون عمودی مرتب می‌شوند، اما ترتیب آن‌ها معکوس است (از پایین به بالا).
محور اصلی: از پایین به بالا.

} container.
; display: flex
;flex-direction: column-reverse
{

نکات:

1-محور اصلی و محور متقاطع:
محور اصلی (Main Axis): جهت مرتب شدن آیتم‌ها، بر اساس مقدار flex-direction است.
محور متقاطع (Cross Axis): عمود بر محور اصلی است.
2-ترکیب با سایر ویژگی‌ها:
می‌توانید از ویژگی‌های دیگر مثل justify-content (برای تراز کردن آیتم‌ها در محور اصلی) و align-items (برای تراز کردن آیتم‌ها در محور متقاطع) همراه با flex-direction استفاده کنید.
3-پشتیبانی مرورگرها:
flex-direction به خوبی در تمام مرورگرهای مدرن پشتیبانی می‌شود.


justify-content

ویژگی justify-content در CSS برای تراز کردن آیتم‌های یک کانتینر فلکس یا گرید در محور اصلی (main axis) استفاده می‌شود. محور اصلی به جهت مرتب‌سازی آیتم‌ها بستگی دارد و توسط ویژگی flex-direction یا grid تنظیم می‌شود.

مقادیر justify-content:

1.flex-start (پیش‌فرض):

آیتم‌ها در ابتدای محور اصلی تراز می‌شوند. فاصله‌ای بین آیتم‌ها ایجاد نمی‌شود.

}container.
;display: flex
;justify-content: flex-start
{

2.flex-end:

آیتم‌ها در انتهای محور اصلی تراز می‌شوند. فاصله‌ای بین آیتم‌ها ایجاد نمی‌شود.

}container.
;display: flex
;justify-content: flex-end
{

3.center:

آیتم‌ها در محور اصلی به صورت وسط‌چین تراز می‌شوند. فاصله‌های مساوی در دو طرف کانتینر قرار می‌گیرد.

}container.
;display: flex
;justify-content: center
{

4.space-between:

آیتم‌ها به گونه‌ای تراز می‌شوند که اولین آیتم در ابتدای کانتینر و آخرین آیتم در انتهای کانتینر باشد. فاصله‌های مساوی بین آیتم‌ها ایجاد می‌شود.

}container.
;display: flex
;justify-content: space-between
{

5.space-around:

آیتم‌ها با فاصله‌های مساوی از یکدیگر تراز می‌شوند. فاصله‌ای که در دو طرف آیتم‌ها قرار می‌گیرد برابر است، اما فاصله‌ی بین آیتم‌ها دو برابر فاصله‌ی کناره‌ها است.

}container.
;display: flex
;justify-content: space-around
{

6.space-evenly:

آیتم‌ها با فاصله‌های کاملاً مساوی از یکدیگر و از کناره‌های کانتینر تراز می‌شوند.

}container.
;display: flex
;justify-content: space-evenly
{

7.start و end:

تراز آیتم‌ها به ترتیب در ابتدای یا انتهای محور اصلی است. این مقادیر بیشتر در گرید و محیط‌های چندزبانه مفید هستند.

} container.
;display: flex
;justify-content: start/end
{

8. left و right:

آیتم‌ها به ترتیب به سمت چپ یا راست کانتینر تراز می‌شوند. این مقادیر نیز بیشتر در گرید استفاده می‌شوند.

} container.
; display: flex
; justify-content: left/right
{


نکات:

1.ارتباط با محور اصلی:
محور اصلی (main axis) به مقدار ویژگی flex-direction بستگی دارد. برای مثال:
اگر flex-direction: row باشد، محور اصلی افقی است.
اگر flex-direction: column باشد، محور اصلی عمودی است.
2.ترکیب با align-items:
برای تراز کردن آیتم‌ها در محور متقاطع (Cross Axis)، از ویژگی align-items استفاده می‌شود.
3.پشتیبانی مرورگرها:
ویژگی justify-content به خوبی در تمام مرورگرهای مدرن پشتیبانی می‌شود.


align-items:

ویژگی align-items در CSS برای تراز کردن آیتم‌ها در محور متقاطع (Cross Axis) در یک کانتینر فلکس (Flexbox) یا گرید (Grid) استفاده می‌شود. محور متقاطع عمود بر محور اصلی است، که جهت آن با ویژگی flex-direction یا تنظیمات گرید تعیین می‌شود.

مقادیر align-items

1.stretch (پیش‌فرض):

آیتم‌ها به گونه‌ای تراز می‌شوند که ارتفاع یا عرض آن‌ها (در محور متقاطع) کشیده شده و کل فضای محور متقاطع را پر کند. برای کار کردن، آیتم‌ها نباید اندازه‌ای ثابت (مثل height یا width) داشته باشند.

} container.
; display: flex
;align-items: stretch
{

2.flex-start:

آیتم‌ها در ابتدای محور متقاطع تراز می‌شوند.

} container.
; display: flex
;align-items: flex-start
{

3.flex-end:

آیتم‌ها در انتهای محور متقاطع تراز می‌شوند.

}container.
; display: flex
;align-items: flex-end
{

4.center:

آیتم‌ها در محور متقاطع وسط‌چین می‌شوند.

}container.
;display: flex
;align-items: center
{

5.baseline:

آیتم‌ها بر اساس خط پایه متنی (Baseline) داخلی‌شان تراز می‌شوند.

} container.
;display: flex
;align-items: baseline
{

نکات

1-ارتباط با محور متقاطع: محور متقاطع عمود بر محور اصلی است: اگر flex-direction: row باشد، محور متقاطع عمودی است. اگر flex-direction: column باشد، محور متقاطع افقی است.
2-تفاوت با justify-content:
align-items: آیتم‌ها را در محور متقاطع تراز می‌کند. justify-content: آیتم‌ها را در محور اصلی تراز می‌کند.
3-ترکیب با align-self: از ویژگی align-self می‌توانید برای تراز کردن یک آیتم خاص به صورت جداگانه استفاده کنید.
4-پشتیبانی مرورگرها: align-items به خوبی در تمام مرورگرهای مدرن پشتیبانی می‌شود.


flex-shrink:

ویژگی flex-shrink در CSS تعیین می‌کند که یک آیتم فلکس (Flexbox) تا چه حد مجاز است کوچک شود اگر فضای موجود در کانتینر کمتر از مقدار لازم برای نمایش تمام آیتم‌ها باشد. این ویژگی بخشی از مدل فلکس‌باکس است و برای کنترل رفتار آیتم‌ها در هنگام کمبود فضا استفاده می‌شود.

مقدار این ویژگی یک عدد غیرمنفی است (پیش‌فرض: 1). این عدد نشان‌دهنده‌ی نسبت کوچک شدن آیتم نسبت به آیتم‌های دیگر در کانتینر فلکس است.
مقادیر معمول:
1) 1 (پیش‌فرض)
آیتم می‌تواند کوچک شود.
تمام آیتم‌ها به طور مساوی کوچک می‌شوند (اگر مقدار پیش‌فرض برای همه یکسان باشد).
2)0
آیتم اجازه ندارد کوچک شود، حتی اگر فضای کانتینر محدود باشد.
به این ترتیب، آیتم اندازه اصلی خود را حفظ می‌کند (مگر اینکه سایر آیتم‌ها کوچک شوند).
3) اعداد بیشتر از 1
آیتم بیش از سایر آیتم‌ها کوچک می‌شود.
نسبت کوچک شدن به مقدار این عدد بستگی دارد.


flex-wrap:

ویژگی flex-wrap در CSS تعیین می‌کند که آیتم‌های یک کانتینر فلکس (Flexbox) در صورت کمبود فضای موجود، آیا باید در یک خط بمانند یا به خطوط جدید منتقل شوند.


flex-grow:

این ویژگی در css برای کنترل رفتار چیدمان آیتم ها در یک کانتینر flex-box استفاده می شود. این خاصیت مشخص می کند که آیا آیتم های یک کانتینر در یک خط واحد قرارگیرند یا در خطوط جدید شکسته شوند.

مقادیر:

0: آیتم ها هیچ فضایی برای رشد ندارند

1(پیشفرض):آیتم ها فضای اضافی را به طور یکسان تقسیم می کنند.

عدد بزرگتر:آیتم های که مقدار flex-grow بیشتری دارند فضای اضافی بیشتری می گیرند.

order:

این ویژگی در css برای ترتیب نمایش آیتم های فرزند در یک کانتینر flex-box یا grid استفاده می شود این خاصیت کنترل میکند هر آیتم در چه جایگاهی در محور اصلی قرار بگیرد صرف نظر از ترتیب آن در کد html. 0 مقدار پیش فرض است. مقدار <number> می تواند مثبت یا صفر باشد مقادیر کمتر زودتر نمایش داده می شود مقادیر بالاتر دیر تر نمایش داده می شود.

نکات:

خاصیت order برای زمانی مفید است که می خواهید ترتیب نمایش را تغییر دهید بدون انکه نیاز باشد ترتیب آیتم ها در کد html تغییر کند. ترتیب آیتم‌ها توسط order در محور اصلی کانتینر که با flex-box مشخص می شود اعمال می گردد و می توانید از order همراه با flex-box و align-self استفاده کنید تا کنترل بیشتری بر ترتیب و اندازه آیتم‌ها،داشته باشید.

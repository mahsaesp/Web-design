آشنایی با git در vscode

Gitچیست؟

گیت (Git) یک سیستم کنترل نسخه توزیع‌شده (Distributed Version Control System) است که برای مدیریت و پیگیری تغییرات در کدهای پروژه‌ها و فایل‌ها استفاده می‌شود. گیت به شما این امکان را می‌دهد که تاریخچه تغییرات را نگهداری کرده و به راحتی تغییرات مختلف را مشاهده و مدیریت کنید. این ابزار برای پروژه‌های نرم‌افزاری و کدنویسی بسیار مفید است، زیرا امکان همکاری میان چندین توسعه‌دهنده را فراهم می‌کند و به هرکسی این اجازه را می‌دهد که به‌طور مستقل روی پروژه کار کند و سپس تغییرات خود را با یکدیگر ترکیب کنند.

مفاهیم اولیه گیت در VSCode:

1.مخزن (Repository): گیت پروژه‌ها را در "مخزن‌ها" ذخیره می‌کند. مخزن گیت معمولاً در پوشه پروژه قرار دارد و شامل تاریخچه تمام تغییرات است.
2.کمیته (Commit):هر بار که تغییرات مهمی در پروژه می‌دهید، یک کمیته جدید ایجاد می‌کنید که به عنوان یک نقطه بازگشت عمل می‌کند.
3.شاخه (Branch):گیت به شما اجازه می‌دهد که پروژه را در چندین شاخه مختلف توسعه دهید.
4.پوش (Push) و پول (Pull): برای همگام‌سازی تغییرات با مخزن از راه دور (مانند GitHub) از دستورات push و pull استفاده می‌شود.

دستورات گیت در ترمینال VSCode

یجاد مخزن گیت جدید: اگر پروژه شما هنوز به گیت متصل نیست، می‌توانید با استفاده از ترمینال داخل VSCode یا با انتخاب گزینه Initialize Repository در بخش گیت (Git) که در نوار کناری VSCode قرار دارد، یک مخزن جدید بسازید در نوار کناری VSCode، روی آیکون Source Control (علامت گیت) کلیک کنید. گزینه Initialize Repository را انتخاب کنید.
الف)مشاهده تغییرات:پس از تغییرات در فایل‌ها، VSCode به طور خودکار آن‌ها را شناسایی می‌کند و در نوار کناری گیت نمایش می‌دهد. فایل‌های تغییر یافته با رنگ‌های مختلف نشان داده می‌شوند سبز برای فایل‌های جدید (Untracked)،زرد برای فایل‌هایی که تغییر کرده‌اند (Modified)
ب)اضافه کردن تغییرات (Stage Changes): برای آماده کردن تغییرات جهت ثبت (commit)، باید تغییرات را stage کنید. در نوار کناری گیت، کنار هر فایل تغییر یافته یک علامت + وجود دارد. با کلیک روی این علامت، فایل به stage اضافه می‌شود.
ج)ثبت تغییرات (Commit): بعد از stage کردن تغییرات، باید آن‌ها را ثبت کنید در بخش گیت، در کادر Message یک توضیح برای تغییرات وارد کنید روی علامت تیک کلیک کنید تا تغییرات ثبت شوند

در VSCode، می‌توانید از ترمینال داخلی برای اجرای دستورات گیت استفاده کنید. برای باز کردن ترمینال: از منوی Terminal گزینه New Terminal را انتخاب کنید. دستورهای گیت را در این ترمینال وارد کنید.

دستورات گیت متداول:

1-بررسی وضعیت گیت (Git Status): این دستور نشان می‌دهد که چه فایل‌هایی تغییر کرده‌اند و آیا فایل‌ها staged شده‌اند یا خیر.
git status

2-اضافه کردن فایل‌ها به stage:برای اضافه کردن یک فایل خاص به stage
git add filename
برای اضافه کردن همه فایل‌های تغییر یافته
git add

3-ثبت تغییرات (Commit): برای ثبت تغییرات با پیام توضیحی
"پیام تغییرات" git commit -m

4-مشاهده تاریخچه تغییرات (Log): برای مشاهده تاریخچه کمیته‌ها
git log

5-ایجاد یک شاخه جدید (Branch): برای ایجاد یک شاخه جدید و تغییر به آن
git checkout -b new-branch

6-انتقال به شاخه دیگر (Switch Branch):برای انتقال به یک شاخه موجود
git checkout branch-name

7-ادغام تغییرات (Merge):برای ادغام تغییرات یک شاخه به شاخه فعلی
git merge branch-name

8-پوش (Push) تغییرات به مخزن از راه دور:برای ارسال تغییرات به مخزن از راه دور (مثلاً GitHub)
git push origin branch-name

9-پول (Pull) تغییرات از مخزن از راه دور: برای دریافت تغییرات جدید از مخزن از راه دور
git pull origin branch-name

مثال کلی:

# 1. اگر پروژه شما هنوز گیت ندارد، مخزن گیت را راه‌اندازی کنید:
git init
# 2. بررسی وضعیت تغییرات پروژه:
git status
# 3. اضافه کردن تغییرات به stage:
git add .
# 4. ثبت تغییرات (commit):
git commit -m "Your commit message"
# 5. مشاهده تاریخچه commit ها:
git log
#6. برای ارسال تغییرات به مخزن از راه دور (مانند GitHub):
git push origin main
#7. برای دریافت تغییرات از مخزن از راه دور:
git pull origin main
